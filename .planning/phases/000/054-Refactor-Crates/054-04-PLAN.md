<action>
Add explicit runtime placement and shard execution surfaces under the
aggregators crate, then rewire validator, watcher, and node integration to
consume those runtime boundaries without turning placement metadata into public
truth. `AggregatorId`, `ShardPlacementTable`, standby metadata, and
`ShardExecutor` remain runtime-operational only. Validators and watchers may
observe or consume those surfaces, but they must not become planner authority,
route authority, or verifier-visible truth owners. `z00z_rollup_node` must stay
the composition root that wires the runtime services together. Rows in the
validator, watcher, and node tables that stay unchanged are still validation
anchors in this slice and must not drift semantically while placement moves.
</action>
<acceptance_criteria>
- Runtime placement and shard execution have explicit module homes.
- Validators and watchers read runtime surfaces without taking over routing or
  planner authority.
- `z00z_rollup_node` still reads as the top-level orchestration root.
- Node theorem validation and the unchanged runtime table rows remain coherent
  with the new placement boundary.
</acceptance_criteria>
