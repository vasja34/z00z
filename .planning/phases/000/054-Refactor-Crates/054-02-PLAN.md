# 054-02 Plan: Backend Extraction And `SettlementStore` Rewire

<objective>
Move RedB/common/memory implementation details onto the new backend seam and
rewire `SettlementStore` to use that seam without changing the live semantic
contract.

Purpose: separate durable adapter wiring from settlement semantics.
Output: backend-specific and backend-neutral helper modules plus a
`SettlementStore` that no longer depends directly on a concrete `RedbBackend`
field or constructor shape.
</objective>

<action>
Extract the concrete RedB implementation into `backend/redb/*`, move
backend-neutral helper code into `backend/common/*` and `backend/memory/*`, and
rework `SettlementStore` so the semantic facade depends on the low-level backend
contracts instead of storing a concrete `RedbBackend` as a first-class semantic
field. Preserve the live `SettlementTreeBackend` trait, settlement root and
proof methods, deterministic replay behavior, and query semantics. This slice
must wire manifest-level backend feature truth for `redb` and `memory`, must
not add `rocksdb`, must not rename public runtime/storage symbols, and must not
weaken the compatibility gates frozen in `054-01`.
</action>

  without tested behavior; the plan forbids it in this wave.
