# 055-01 Plan: Batch Proof Wire Contract And Deterministic Codec

<objective>
Freeze the exact `BatchProofBlobV1` wire contract and land its deterministic
codec on the live storage seam without changing the existing `ProofBlob`

depend on one canonical byte format instead of an evolving in-memory sketch.

canonical export path, and guardrails that keep `ProofBlob` unchanged.

<action>
Introduce the public storage-owned batch-proof wire contract:
`BatchProofBlobV1`, `BatchProofHeaderV1`, `BatchPathEntryV1`,
`BatchProofLimits`, `WitnessNodeV1`, `OpeningEntryV1`, the frozen
family-specific opening payload structs, and `PathWitnessRefV1`. Encode and
decode them as one positional deterministic binary contract with exact field
order, explicit big-endian scalar rules, and pre-allocation bounds checks.
Add explicit wire-tag enums for proof family, opening kind, node domain,
sibling side, and leaf-family mapping so the live internal
`SettlementLeafFamily::{Terminal, Right}` surface can stay unchanged while the
wire contract uses the upgrade paper's public naming rules. Export the new
types through one canonical `z00z_storage::settlement` path, keep
`ProofBlob` unchanged, and add guardrails that fail if the repository starts
presenting `BatchProofBlobV1` as a replacement rather than an additive
envelope. In this slice, support only the current live non-sharded generation
and reject any partial shard context or future generation claim.
</action>
