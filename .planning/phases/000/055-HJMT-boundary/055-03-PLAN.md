`ProofBlob` and `settlement_proof_blobs` intact.

`SettlementStore` method that returns `BatchProofBlobV1` for a set of
`SettlementPath` values, while keeping `settlement_proof_blobs` untouched as

consuming the already-live single-path proof context owned by storage and

engine or overload the old batch helper to return a different type.

  builder must derive from existing proof contexts.

  provenance; regeneration commands and re-encode checks are mandatory.
- Downgrade: the new batch API can silently replace the old independent
  baseline; compatibility tests keep all three surfaces visible.
</threat_model>
