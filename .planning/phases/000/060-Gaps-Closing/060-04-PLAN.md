<action>
Implement `A2` through `A5` exactly. Move `RightClassConfig`,
`RightsConfigEntry`, and rights-config parsing or loader ownership under
`crates/z00z_core/src/rights/`. Update internal imports across the repository
to the new owner path. Keep `assets/right_config.rs` only as a documented
compatibility shim if public compatibility requires it; otherwise retire it.
Demote `assets` shim-first imports for actions, policies, and vouchers so new
code and docs use the owner modules directly. The exact `action_pool`,
`policy_descriptor`, and `voucher_config` shims under `assets/` must be treated
as compatibility-only surfaces rather than preferred authoring paths. Remove
the default `rights`
bootstrap implication from `src/assets/assets_config.yaml` and stop the
repository from treating `assets_config.yaml` and `genesis_config*.yaml` as
co-equal authority stories, or clearly relabel the mixed asset file as
compatibility or fixture-only if it must temporarily survive. Synchronize
README, module docs, examples, and fixtures to the intended owner boundary. Do
not add `actions_config.yaml`, `policies_config.yaml`, `vouchers_config.yaml`,
or `rights_config.yaml`. Do not fork or relocate the live
`z00z_utils::codec::canonical_json.rs` owner, do not rename or split
`genesis_policies.rs`, and do not collapse the storage-owned boundary in
`crates/z00z_storage/src/settlement/leaf.rs` by importing
`z00z_core::assets::AssetLeaf` as the storage semantic owner.
</action>
