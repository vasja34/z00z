<action>
Implement the `B1` and `B2` contract freeze. Make the live process model
explicit in docs, config parsing, and topology tests: one aggregator equals one
OS process by default and may own multiple primary shards. Add a YAML-selectable
`execution.shard_mapping` contract with default `aggregator_owned` and opt-in
`shard_process`. Reject mixed or invalid profiles, reject multi-primary owners
in `shard_process`, preserve process-scoped `journal_path` lineage validation in
`aggregator_owned`, and record the selected mapping in the HJMT evidence path.
The canonical operator-facing default must be spelled out literally as:

```yaml
execution:
  shard_mapping: "aggregator_owned"
```

`shard_process` is valid only as an explicit opt-in override, never as an
implied default. Keep one aggregator equals one OS process by default as the
live contract, while allowing many shards per aggregator process under
`aggregator_owned`.
Keep the later implementation shape small and explicit around a local
`AggExecutionCfg` plus `ShardMapping` enum instead of widening topology truth.
Generate one `shard_process` fixture home through
`crates/z00z_rollup_node/tests/support/test_hjmt_home.rs` or an adjacent
temporary-fixture helper instead of creating a second checked-in canonical
runtime home. Do not promote `shard_process` to the production default, and do
not fork the checked-in `sim_5a7s` authority or the canonical `SIM-5A7S`
default fixture beyond making its default `aggregator_owned` mapping explicit.
</action>
