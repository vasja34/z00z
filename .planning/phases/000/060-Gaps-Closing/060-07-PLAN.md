<action>
Implement `D1`, `D2`, and `D5` exactly. Publish one repository-owned MVP
profile matrix that covers at least `fee_credit_v1`, `service_entitlement_v1`,
`data_access_v1`, `agent_budget_v1`, `validator_mandate_lock_v1`, and
`transferable_claim_v1`, with family, actions, policy surfaces, lifecycle, and
fail-closed rules. Then define wallet projection semantics per profile across
`wallet.object.*`, `wallet.asset.*`, quarantine state, spendability, RPC
projection, persistence fields, and backup roundtrip semantics. Keep
`wallet.asset.*` cash-only. `wallet.asset.*` remains cash-only and must not
project rights or vouchers as spendable value. Keep `.wlt` and
`WalletExportPack` as the only wallet-local authorities, and do not introduce a
second wallet DB or a second transfer bundle. Each profile row must state
whether the profile id is already live or is only a proposed Phase 060 catalog
identifier. Bind live anchors explicitly where they exist: `service_entitlement_v1` to live
`service_entitlement`, `data_access_v1` to live `data_access`,
`validator_mandate_lock_v1` to live `validator_mandate`, `agent_budget_v1` to
live `machine_capability` plus `one_time_use`, and `fee_credit_v1` plus
`transferable_claim_v1` to the existing voucher object model and the cited
product docs. Preserve cash-only asset-row persistence through
`wallet_asset_store`, and require any profile whose policy availability is not
`Available` to remain quarantined instead of surfacing as ordinary balance.
Unknown-policy objects remain in durable quarantine until a known `Available`
policy verdict exists.
Do not present proposed profile ids as already-live code identifiers.
</action>
<acceptance_criteria>

- each profile has actions, policy surfaces, lifecycle, and fail-closed rules.
- each profile row states whether its identifier is live or proposed and cites
  the exact live anchor when one exists.
- wallet projection semantics are explicit for `wallet.object.*`,
  `wallet.asset.*`, quarantine, and spendability.
- `wallet.asset.*` remains cash-only.
- rights and vouchers are not projected as cash balances.
- asset rows continue using cash persistence only, and non-`Available` policy
  states remain quarantined.
- unknown-policy objects remain in durable quarantine.
- `.wlt` and `WalletExportPack` remain the only wallet-local authority plane.
</acceptance_criteria>
