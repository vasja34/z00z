# 060-15 Plan: MVP Closeout For Refund Source Binding, One-Plane Issue/Create, Incomplete Publication States, And Monotonic Right Delegation

<objective>
Close the remaining MVP-required Phase 059 gaps revalidated against the live
codebase by binding voucher reject or refund outputs to both refund target and
restricted source context, widening the existing object-package RPC to cover
voucher issue and right create from truthful source or create context, making
`VerdictKind::Incomplete` plus `AlertKind::ValidatorIncomplete` live runtime
states on the current publication path, and proving that right delegation
cannot widen authority on the existing transfer contract.
</objective>

<coverage_contract>
This plan closes exactly four MVP-required live gaps:

1. voucher reject or refund outputs must bind to the declared
   `refund_target_commitment`, and restricted backing must also bind to the
   declared source context;
2. `wallet.object.preview_package` and `wallet.object.build_package` must be
   able to build `VoucherAction::Issue` and `RightAction::Create` packages from
   truthful source or create context without a parallel RPC surface;
3. validator and watcher runtime surfaces must emit real
   `VerdictKind::Incomplete` and `AlertKind::ValidatorIncomplete` states on the
   existing publication contract for retry-pending, missing-artifact, and
   publication-gap cases;
4. the live `delegate_right` contract must not widen authority. Monotonic
   attenuation must be enforced on the existing `RightAction::Transfer` path
   for the fields the current `RightLeaf` contract actually represents, and any
   unsupported widening semantics must fail closed instead of being overclaimed.

This slice supersedes the overlapping `060-14` execution scope for these exact
gaps. It must not add a second wallet builder, a second publication checker, a
parallel right hierarchy, or a second canonical Phase 060 task id.
</coverage_contract>
