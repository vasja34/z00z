## Shared Validation Rules

| Rule | Requirement |
| --- | --- |
| Fail-fast gate | Run `./.github/skills/smart-tests-bootstrap/scripts/bootstrap_tests.sh` first on every step. If it fails, stop, fix, and rerun it before broader validation. |
| Broad Rust gate | Run `cargo test --release` for any Rust, test, simulator, serialization, public API, or verification-script affecting slice. If a slice is docs-only or report-only, explicitly record why the broad Rust gate was not required. |
| Review repetition | Run `/GSD-Review-Tasks-Execution` in YOLO mode at least 3 times and only stop after at least 2 consecutive runs show no significant issues. |
| Commit discipline | If a step needs a commit after validation, use `/z00z-git-versioning`. |
| Evidence discipline | Preserve the exact artifact or log that justifies the pass signal; do not replace it with prose-only claims. |

## Global Reject Conditions

| Condition | Why it fails the phase |
| --- | --- |
| `shard_process` becomes the default without a successful `060-S10` A/B packet | It violates the carried production-default guard and turns an option into an unjustified semantic change. |
| A throughput claim is made from worker-local metrics only | It breaks the durable-publication measurement contract. |
| Docs, supply-chain, bench, or closeout work creates a second authority store | It introduces concept drift and makes later verification non-comparable. |
| Wallet changes widen `.wlt`, `WalletExportPack`, or `wallet.asset.*` into a second authority plane | It breaks `D5` and invalidates the one-plane wallet contract. |
| Adversarial highs are closed by prose without a reproducer, disprover, or linked closure memo | It leaves the security packet untestable and unowned. |
| C4 runtime wins are achieved by skipping security or semantics-preserving gates | It produces false performance wins and invalidates final closure. |
