<action>
Define the exact runtime-owned route-table contract, including canonical
ordering, digest recomputation, routing-generation binding, and full-range
coverage checks. Extend the planner path so central and per-aggregator modes
produce the same accepted `BatchPlanned` digests and the same reject matrix on
identical inputs. Keep cross-shard work rejected before semantic execution and
do not move planner authority into storage, validators, or simulator code.
</action>

- Cross-shard batches reject before semantic handoff.
- No second planner authority surface is introduced.
</acceptance_criteria>
