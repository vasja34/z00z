## Tests

### 1. Canonical Definition Identity

expected: When you exercise a config, registry, or genesis path that rebuilds an asset definition, caller-chosen placeholder ids do not survive. The resulting definition uses one canonical id derived from the payload, and changing identity-bearing fields causes either a different id or an explicit rejection.
result: [pending]

### 2. Registry Snapshot Tamper Rejection

expected: A snapshot with the same asset ids but changed canonical payload content is rejected on apply, and concurrent snapshot publication does not let an older accepted version overwrite a newer one.
result: [pending]

### 3. Protected Genesis Fail-Closed

expected: Protected-network genesis rejects missing anchors, mismatched anchors, unknown chain types, and weak seeds instead of degrading into devnet-like success paths.
result: [pending]

### 4. Wire And DTO Boundary

expected: Public DTO or wire import rejects secret-bearing payloads, preserves supported protocol-state flags, and a rebuilt asset must pass full cryptographic verification instead of only structural validation.
result: [pending]

### 5. Owner And Fee Authority

expected: Mutating owner or stealth-critical fields after signing breaks authoritative verification, and a non-native Coin-class asset is rejected when used as the fee asset.
result: [pending]

### 6. Nonce And Amount Policy

expected: Production nonce helpers fail closed on time-provider errors, and amounts outside the configured proof-width policy are rejected instead of being accepted with implicit fallback behavior.
result: [pending]
