<action>
Flatten the internal service shard trees so both `services/app/*` and
WalletService implementation files live at one-level `src/services/*.rs` paths,
while keeping `services::WalletService`, `app_service.rs`, and related
re-export surfaces stable. Rewrite `#[path]` and `include!` boundaries using
the live shard topology (`services/app/*`, `wallet/actions/*`,
`wallet/session/*`, `wallet/store/*`) rather than the stale TODO wrapper
assumptions, and update source-anchor-sensitive tests that read service shard
files by relative path. Do not mix the RPC method implementation churn into
this slice.
</action>

- Source-anchor checks in `test_app_service_suite.rs` and service tests point at
  the new canonical files.

  wrappers remain in the executing tree.
</acceptance_criteria>
