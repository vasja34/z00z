<action>
Move the support-side RPC tree into one-level `src/rpc/*.rs` files: facade
root, logging, DTO/types, error mapping, dispatcher helpers, and dispatcher
wiring. Preserve the old import surface through `adapters::rpc`, ideally by
pointing `src/adapters/mod.rs` at `#[path = "../rpc/mod.rs"] pub mod rpc;`, or
an equivalent thin facade declaration that exposes no new crate-root `rpc`
module. In the same slice, move `wallet_config.yaml` out of `src/`, update
every embedded or runtime loader anchor that depends on it, and keep a single
canonical default config source for wallet paths and RPC logging. This
ownership includes the embedded source test in
`services/wallet/tests/test_wallet_paths_suite.rs` and the RPC logging helper
coverage in `tests/test_common/test_rpc_logger.inc`. Leave RPC method
implementation files for the next method-specific wave.
</action>
<acceptance_criteria>
- RPC support, logging, types, and dispatcher files live in one-level
  `src/rpc/*.rs` locations.
- `adapters::rpc` remains the stable compatibility entrypoint.
- `wallet_config.yaml` no longer lives under `src/`, and all embedded or
  runtime loaders point to the new canonical location.
- The embedded config source test in `test_wallet_paths_suite.rs` points at the
  same new canonical location.
- `test_rpc_logger.inc` still exercises the preserved `adapters::rpc::logging`
  surface against that same canonical config source.
- No method-implementation rename debt is mixed into this slice beyond import
  updates required to keep the facade compiling.
</acceptance_criteria>

<threat_model>
- Moving the wallet config without updating every embedded-loader site would
  create a hidden split authority between stale and live defaults.
- Exposing a new crate-root `rpc` API surface would widen public contract
  churn beyond the TODO's compatibility rules.
- Mixing support and method churn would make RPC regressions hard to localize.
</threat_model>

<verification>
This slice is valid only if the RPC support tree is physically flat while the
compatibility facade and config authority remain singular.
</verification>

- `adapters::rpc` remains stable.
- `wallet_config.yaml` has one canonical non-`src/` home.
