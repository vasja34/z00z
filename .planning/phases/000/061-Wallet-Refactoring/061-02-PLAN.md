<action>
Flatten the shared `db` helper subtrees (`index_codecs`, `storage_backend`) to
one-level `src/db/*.rs` files, and rename `redb_wallet_crypto` to the neutral
`wallet_store_crypto*` family under `src/db/`. Update all Rust imports in
wallet, security, backup, wasm, and key-manager code to the new neutral path,
but preserve persisted labels, KDF names, schema text, and domain-separated
strings exactly as they are today. Keep the backend relocation out of this
slice; `db::redb_wallet_store` still points at the old implementation tree
until the next plan owns that seam directly.
</action>
<acceptance_criteria>
- `db/redb_wallet_crypto/*` becomes `db/wallet_store_crypto*` with neutral
  Rust-path naming.
- `db/index_codecs/*` and `db/storage_backend/*` are flattened to one-level
  `src/db/*.rs` files.
- Rust imports no longer depend on `db::redb_wallet_crypto`.
- Persisted labels, schema versions, KDF labels, and
  `z00z.crypto.redb_wallet_crypto...` strings remain unchanged.
</acceptance_criteria>
