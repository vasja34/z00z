```rust
pub trait SpendProofBackend {
    fn suite_id(&self) -> &'static str;

    fn prove(
        &self,
        stmt: &SpendProofStmt,
        wit: &SpendProofWitness,
    ) -> TxVerifierResult<SpendProofArtifact>;

    fn verify(
        &self,
        stmt: &SpendProofStmt,
        proof: &SpendProofArtifact,
    ) -> TxVerifierResult<()>;
}

pub struct SpendProofStmt {
    pub pkg_digest: [u8; 32],
    pub prev_root: [u8; 32],
    pub chain_id: u64,
    pub tx_ver: u16,
    pub input_refs: Vec<SpendInputRef>,
    pub input_leaves: Vec<SpendInputLeaf>,
    pub output_leaves: Vec<AssetLeaf>,
    pub nullifiers: Vec<[u8; 32]>,
}

  pub struct SpendProofWitness {
    pub receiver_secret: ReceiverSecretWitness,
    pub sender_secret_scalar: SenderSecretScalarWitness,
    pub input_witnesses: Vec<SpendInputWitness>,
  }

  pub struct SpendInputWitness {
    pub view_sk: ViewSecretWitness,
    pub k_in: InputKeyWitness,
    pub owner_handle: OwnerHandleWitness,
    pub owner_tag: OwnerTagWitness,
    pub asset_id: AssetIdWitness,
    pub s_in: NullifierSecretWitness,
  }

pub struct SpendProofArtifact {
    pub proof_hex: String,
    pub pub_hash_hex: String,
}
```

`SpendProofStmt` must remain a typed projection of the existing canonical
statement builder in `spend_verification.rs`. `SpendProofWitness` stays
producer-only and must not become a public wire shape.

## Public Inputs Inventory

The backend must bind exactly these public inputs, in canonical order:

1. `build_tx_package_digest(...)` output as the only public and persisted root.
2. Typed `prev_root` from the same checkpoint pipeline used today.
3. Explicit scope tuple: `chain_id`, `chain_type`, `chain_name`, and `tx_ver`.
4. Ordered `SpendInputRef` vector.
5. Ordered `SpendInputLeaf` public fields, including the state-key binding now
   carried by `input_asset_id_hex`.
6. Ordered output `AssetLeaf` public fields already rechecked by
   `verify_self_decrypt()`.
7. Fee-as-output balance data required by the live verifier boundary.
8. The regular-spend nullifier vector.
9. Explicit range-proof mode/commitment compatibility inputs unless those
   checks are migrated into the backend with a spec-backed compatibility note.

## Private Witness Inventory

The backend must prove, but must not publish, these witness-only relations:

1. `ReceiverSecret`.
2. The sender-side secret scalar or equivalent witness that explains `r_pub`.
3. The live owner/view chain already enforced by `verify_spend_rules(...)`,
   including `view_sk`, `k_in`, `owner_handle`, `owner_tag`, and `asset_id`.
4. `s_in` or equivalent nullifier-secret material.
5. Any additional witness needed to preserve the current ordered theorem shape
   without moving private material into the public carrier.

## Closure Rule

The plan must treat the following sentence as the authoritative closure rule
for every downstream task in this wave:

> Full theorem closure does **not** mean recovering the full
> `verify_spend_rules(...)` theorem from the current public-only carrier fields
> alone. It means recomputing the canonical public statement from public tx
> facts and verifying a theorem-carrying proof that attests to the witness-only
> relations the public carrier cannot reveal.
