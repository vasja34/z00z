pub struct SpendProofWitness;

pub struct StatementBoundSpendProofBackend;

impl SpendProofBackend for StatementBoundSpendProofBackend {
    fn prove(
        &self,
        stmt: &SpendProofStmt,
        _wit: &SpendProofWitness,
    ) -> Result<SpendProofArtifact, SpendProofBackendError> {
        Ok(self.encode_artifact(stmt))
    }

    fn verify(
        &self,
        stmt: &SpendProofStmt,
        proof: &SpendProofArtifact,
    ) -> Result<(), SpendProofBackendError> {
        let proof_bytes = decode_canonical_hex(&proof.proof_hex)?;
        let (stmt_hash, pub_hash) = decode_artifact_bytes(&proof_bytes)?;

        }
        let expected_pub_hash = stmt.public_hash();
        if expected_pub_hash != pub_hash {
            return Err(SpendProofBackendError::PublicHashMismatch);
        }
        Ok(())
    }
}

```rust
pub fn verify_spend_rules(stmt: &SpendStmt) -> Result<(), SpendRuleErr> {
    let owner_handle = derive_owner_handle(&stmt.receiver_secret);
    let view_sk = derive_view_secret_key(&stmt.receiver_secret)
        .map_err(|_| SpendRuleErr::BadView)?;

    for (index, input) in stmt.spend_ins.iter().enumerate() {
        let r_pub = decode_r_pub(&input.r_pub_in)
            .map_err(|_| SpendRuleErr::BadRPub { index })?;
        let dh = compute_dh_receiver(&view_sk, &r_pub)
            .map_err(|_| SpendRuleErr::BadDh { index })?;
        let k_in = derive_k_dh(&dh);
        let expected_tag = compute_owner_tag(&owner_handle, &k_in);
        if expected_tag != input.owner_tag_in {
            return Err(SpendRuleErr::BadOwnerTag { index });
        }

        let expected_leaf_ad_id = hash_zk::<AssetIdDomain>("", &[&input.s_in]);
        if expected_leaf_ad_id != input.leaf_ad_id_in {
            return Err(SpendRuleErr::BadAssetId { index });
        }

        let expected_nullifier = derive_spend_nullifier(input.chain_id, &input.s_in);
        if expected_nullifier != input.nullifier_in.ok_or(
            SpendRuleErr::MissingNullifier { index }
        )? {
            return Err(SpendRuleErr::BadNullifier { index });
        }
    }

    if !verify_balance_eq(&stmt.spend_ins, &stmt.c_outs) {
        return Err(SpendRuleErr::BadBalance);
    }
    if !stmt.range_ok {
        return Err(SpendRuleErr::BadRange);
    }
    Ok(())
}
```

## Final Closure Definition

This plan uses one non-negotiable closure rule:

> Final closure does **not** mean reconstructing the full
> `verify_spend_rules(...)` theorem from the current public-only carrier.
> Final closure means recomputing the canonical public statement from public tx
> facts and verifying a theorem-carrying proof over the witness-only relations
> that the public carrier cannot reveal.

The phase is not closed until this sentence is true in code, tests, integrity
ledgers, closeout ledgers, validation status, and stage-surface wording.

## Frozen Theorem Contract

The backend introduced by this plan must prove the existence of witness
`W = {receiver_secret, input_s_in[0..n)}` for the public statement
`S = {pkg_digest, prev_root, chain scope, ordered input refs, ordered input
leaves, ordered output leaves, ordered nullifiers, range_ok}` such that the
proof verifies the same ordered rule kernel already enforced by
`verify_spend_rules(...)` and enumerated by `spend_order()` /
`spend_triplets()`.

Formal closure target:

```text
Given S and W, prove T(S, W) where:

1. owner_handle = derive_owner_handle(receiver_secret)
2. view_sk = derive_view_secret_key(receiver_secret)
3. for each ordered spend input i:
   3.1 r_pub_i = decode_r_pub(S.input_leaves[i].r_pub)
   3.2 dh_i = compute_dh_receiver(view_sk, r_pub_i)
   3.3 k_in_i = derive_k_dh(dh_i)
   3.4 compute_owner_tag(owner_handle, k_in_i)
       == S.input_leaves[i].owner_tag
   3.5 hash_zk::<AssetIdDomain>("", &[W.input_s_in[i]])
       == S.input_leaves[i].leaf_ad_id
   3.6 derive_spend_nullifier(S.chain_id, W.input_s_in[i])
       == S.nullifiers[i]
4. S.nullifiers are unique in order
5. sum(S.input_commitments) == sum(S.output_commitments)
6. S.range_ok == true
```

Hard consequences of that theorem definition:

- `view_sk`, `owner_handle`, and `k_in` are theorem-internal derivations, not
  extra serialized witness slots.
- No separate sender-side secret-scalar slot is part of the frozen closure
  contract because the current theorem authority does not require it.
- `c_in`, `c_out`, and `range_ok` remain public theorem inputs carried by the
  canonical statement; they are not new hidden witness material.
- Any future desire to prove a stronger sender-secret theorem is a separate
  spec change, not part of this closure wave.

### Frozen Witness-Source Table

| Theorem element | Role | Source or derivation that must be preserved |
| --- | --- | --- |
| `receiver_secret` | private source witness | live on the producer path in Stage 4 as `sender_recv_sec`, then decoded with `ReceiverSecret::from_bytes(...)` |
| `input_s_in[i]` | private source witness | live on the producer path via `resolve_input_pack(...).s_out` / `resolve_input_secret(...)` inside `witness_gate.rs` |
| `owner_handle` | derived theorem value | `derive_owner_handle(receiver_secret)` |
| `view_sk` | derived theorem value | `derive_view_secret_key(receiver_secret)` |
| `k_in[i]` | derived theorem value | `compute_dh_receiver(view_sk, decode_r_pub(r_pub_in))` then `derive_k_dh(...)` |
| `owner_tag_in[i]` | public theorem input | canonical `SpendInputLeaf.owner_tag` / public spend statement |
| `leaf_ad_id_in[i]` | public theorem input | canonical `SpendInputLeaf.leaf_ad_id` / public spend statement |
| `nullifier_in[i]` | public theorem input | canonical ordered nullifier vector in the public spend statement |
| `c_in[i]`, `c_out[j]` | public theorem inputs | canonical input/output commitments already carried by the current public statement |

## Blockers And How To Close Them

### Blocker A: The backend witness surface is landed, but theorem verification is still absent

Current state:

- `SpendProofWitness` now carries the frozen minimal theorem witness
  inventory.
- `build_public_spend_contract(...)`, `witness_gate.rs`, and the Stage 4
  runtime flow now pass real witness material into `backend.prove(...)`.

Why this blocks closure:

- A non-empty witness surface alone still does not attest receiver-secret
  binding, owner/view derivations, `s_in -> leaf_ad_id` binding, or
  `s_in -> nullifier` binding. Those relations are only closed when the
  backend verifies a theorem-carrying proof instead of a statement-bound
  envelope.

Required closure:

- Keep the landed producer-only witness structure frozen and replace the
  statement-bound backend artifact with theorem verification over that same
  witness inventory.

Frozen target shape:

    pub input_s_in: Vec<[u8; 32]>,

```

Implementation note:

  secret-scalar slot.
- The current theorem does **not** require serialized `view_sk`,
  `owner_handle`, or `k_in`; those are deterministically derived inside the
  theorem from `receiver_secret`, `r_pub_in`, and `s_in`.

  `resolve_input_pack(...).s_out`.

    fn prove(
        &self,
        stmt: &SpendProofStmt,
        wit: &SpendProofWitness,

        &self,
        stmt: &SpendProofStmt,
        proof: &SpendProofArtifact,

}
```

Implementation note:

  `BulletproofsPlusTheoremSpendBackend` behind `default_spend_proof_backend()`
  with `suite_id() == "regular_spend_theorem_bpplus"`.
- `StatementBoundSpendProofBackend` may remain only as a legacy regression
  fixture or migration helper and must no longer be the default accepted suite.
- Freeze the artifact bytes exactly as:

```text
proof_hex = hex(

  | suite_id_len:u8
  | suite_id_utf8
  | stmt_hash:[32]
  | pub_hash:[32]
  | proof_len:u32_le
  | proof_bytes
)
```

- `pub_hash_hex` remains the canonical mirror of `pub_hash` for wire
  compatibility.
- Introduce explicit backend errors for `LegacyArtifactRejected`,
  `UnsupportedSuite`, `MissingTheoremProof`, and
  `TheoremRelationMismatch` so theorem failures do not collapse into generic
  statement mismatch handling.
- Legacy rejection is exact and fail closed:
  - v1 prefix `z00z.spend.proof.backend.v1` -> reject as legacy
    statement-bound artifact

    unsupported suite

  - proof bytes that do not satisfy `T(S, W)` -> reject as theorem relation
    mismatch
