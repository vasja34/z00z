<action>
Implement D-05 by adding the storage-owned generalized backend trait for settlement operations and replacing live `AssetTreeBackend` usage in place with settlement operations for put, delete, get, lookup, list, prove, apply-batch, and right transition flows for create, transfer, consume, revoke, expire, and challenge. Route `SettlementPath`, `SettlementLeaf`, `SettlementStateRoot`, and `FeeEnvelope` through the existing forest store seam rather than adding a parallel runtime. Keep the path index internal and rebuildable, add typed errors for unsupported generation, wrong leaf family, wrong proof family, and wrong fee binding, remove or fail closed backend mode names that select Phase 052 compatibility, old forest, or dual-verify lanes, and prevent serialization from projecting generalized stores into compatibility artifacts. Preserve asset leaf handling only as `SettlementLeaf::Asset`, not as a separate public root/runtime authority.
</action>
