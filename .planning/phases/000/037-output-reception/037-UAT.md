### 1. Canonical Range Receive Lane

expected: The implemented Phase 037 receive flow should still have exactly one canonical range lane. A real range receive run resumes through `WalletService::recv_range(...)`, keeps progress in `ScanStatePayload`, uses `StealthOutputScanner` for ownership detection, and persists owned claims only through the explicit `recv_route(..., PersistClaim)` gate. No second range receive authority should appear.
