<context>
Phase 051 already established the facade direction and explicit compatibility
backend. This supplemental plan is a bounded readiness gate: it does not ship
the physical forest backend, but it must prove that all remaining caller-facing
authority and verification seams are stable enough that Phase 052 can begin as
full HJMT implementation on one canonical semantic boundary.

This plan is successful only if Phase 052 can begin from one caller-facing
seam. If any task in this plan fails, execution must stay in Phase 051 and the
failure must be recorded as a blocker instead of opening full forest backend
implementation work early.
</context>

<exit_gate>
Phase 052 may begin only when all of the following are true:

1. The facade exists and is real, not decorative.
2. The compatibility backend is the only legacy execution path behind the
   facade.
3. Root vocabulary is locked.
4. The proof envelope is locked.
5. Downstream authority leakage is closed.
6. The compatibility golden semantic corpus is green.
7. The reload, checkpoint, and proof reject corpus is green.

Until those seven conditions are satisfied, Phase 052 remains blocked.
</exit_gate>
