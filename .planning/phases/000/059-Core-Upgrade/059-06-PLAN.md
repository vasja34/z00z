<action>
Add a typed object package envelope for runtime ingress that carries object
family, selected action, policy descriptor hash, action pool id, required
rights, object witnesses, typed storage delta references, fee envelope reference,
prior root, and expected new root. Aggregators may route, batch, and carry this
envelope, but must not evaluate policy semantics beyond routing preconditions.
Validators must verify object presence, policy/template match, action
membership, required rights and scope, signatures/control keys, verifier-safe
attestations, quota/expiry, typed deletion/creation/update, conservation, and
fee support separation. Add precise verdict codes for unknown policy, invalid
backing, wrong family proof, voucher used as cash, right used as value, missing
right, out-of-scope right, expired/revoked/consumed right, replay, double
redeem, residual mismatch, forced acceptance, stale root, and fee boundary
violation. Watchers must emit matching alert records and rollup publication
status must surface object-family verdict evidence where checkpoint verification
depends on it.
</action>
<acceptance_criteria>
- Aggregator ingress and batch planning can carry typed object package metadata
  without becoming semantic authority.
- Validator verdicts are specific enough to distinguish unknown policy, invalid
  backing, wrong family proof, missing/expired/revoked/consumed right, replay,
  double redeem, forced acceptance, and fee-boundary violations.
- Unknown policy descriptors fail closed at validator level.
- Watchers produce alert records for every mandatory invalid object class.
- Rollup status/RPC exposes object-family verdict/publication evidence without
  exposing wallet secrets.
- Tests prove invalid packages produce no storage root update.
</acceptance_criteria>

<threat_model>
- Aggregators that decide policy semantics can become a duplicate authority.
- Generic invalid verdicts can hide value/authority boundary failures.
- Watchers that do not alert on wrong-family proof, double redemption, or
  value-bearing rights can let consensus-critical failures look silent.
</threat_model>
