<action>
Add family-first scan classification before object-specific payload recovery.
Keep `recv_claim_asset` or its replacement asset-only and add voucher/right
payload opening paths with policy descriptor availability checks. Add wallet
package building for object actions: selected action, live inputs, planned
outputs, required rights, policy descriptors, proofs, signatures, attestations,
prior root, typed deltas, and fee envelope. Reject packages where a voucher is
used as cash input, a right is used as value input, an unknown-policy object is
spendable, or a required right is missing, out of scope, expired, revoked,
consumed, or delegated incorrectly. Preserve `asset.send`, `asset.receive`, and
asset history as cash-only. Add typed object/voucher/right RPC methods or an
object namespace for inventory, quarantine, voucher offer/accept/reject,
voucher redeem/refund/transfer, right grant/delegate/consume/revoke/challenge,
and package preview/build. Extend backup export/import to include typed object
payloads, descriptor refs, openings, and quarantine state without leaking
secrets.
</action>
<acceptance_criteria>
- Wallet scan classifies Terminal, Voucher, and Right leaves before attempting
  payload recovery.
- Unknown policy objects quarantine and never appear as spendable cash.
- Package builder rejects voucher-as-cash, right-as-value, expired/revoked/
  consumed/missing right, stale policy, forced acceptance, and fee boundary
  confusion.
- Existing asset RPC methods remain cash-only and reject voucher/right ids.
- New typed object RPC exposes inventory, lifecycle, package preview/build, and
  quarantine status without overloading asset response types.
- Backup export/import roundtrips typed object payloads and preserves old asset
  backups.
</acceptance_criteria>

<threat_model>
- Overloading asset RPC for vouchers or rights can trick callers into treating
  conditional claims or authority as cash.
- A package builder that trusts wallet state rather than validator-readable
  descriptors can create wallet-only policy truth.
- Backups that omit descriptor refs or quarantine reasons can restore unsafe
  objects as spendable.
</threat_model>

preserves object safety.
