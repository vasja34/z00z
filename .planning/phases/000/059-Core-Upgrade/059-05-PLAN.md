<action>
Add typed storage delta records that explicitly carry deleted objects, created
objects, updated residual objects, attached `FeeEnvelope`, selected action,
policy descriptor hash, prior root, and expected new root. Add voucher lifecycle
operations for issue/create, offer, accept, reject, transfer when policy allows,
full redeem, partial redeem with exact residual voucher value, refund, expiry,
and invalid transition rejection. Add right action handoff compatibility for
grant/create, transfer/delegate, consume, revoke, expire, and challenge while
keeping rights zero-value. Add conservation checks where final value assets and
conditional value vouchers contribute according to their action semantics and
rights contribute zero. Keep wallet secrets/private openings outside storage;
storage validates committed leaves, paths, roots, proof family, policy hashes,
and typed deltas only.
</action>
<acceptance_criteria>
- Delta records list deleted inputs, created outputs, updated residual objects,
  fee envelope, selected action, policy hash, prior root, and expected new root.
- Voucher full redeem deletes voucher and creates asset output with exact
  remaining value.
- Voucher partial redeem creates asset output and residual voucher or explicit
  closure with exact value accounting.
- Double redeem, residual mismatch, unbacked issuance, invalid refund, expired
  voucher use, and forced acceptance fail closed.
- Rights never add to conservation totals and any right-as-value path fails.
- `FeeEnvelope` remains separate support and cannot become voucher backing or
  right authority.
- Store recovery and journal replay preserve typed object deltas.
</acceptance_criteria>

<threat_model>
- Treating a voucher as spendable cash can bypass receiver consent and backing
  checks.
- Treating a right as value can mint hidden budget or fee authority.
- Keeping fees inside voucher or right semantics can hide processing support as
  object state.
- Storage that reads wallet payload secrets becomes a wallet authority instead
  of a settlement authority.
</threat_model>
