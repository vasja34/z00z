<objective>
Move wallet persistence from asset-only storage toward a typed owned-object
inventory with three projections: spendable cash assets, conditional voucher

<action>
Add a wallet owned-object inventory facade. Either add `OwnedObject` as a new
generalized payload kind or add `OwnedVoucher` and `OwnedRight` kinds behind a
shared query facade; do not reuse `OwnedAsset` for vouchers or rights. Preserve
`OwnedAssetPayload::VERSION` and `PAYLOAD_VERSION_OWNED_ASSET = 1` read paths.
Add typed payload envelopes for asset, voucher, and right objects with object
family, object id, policy availability, lifecycle status, private openings,
labels, scan refs, checksum, and quarantine reason. Add indexes by object
family, lifecycle/status, policy availability, and owner/holder where existing
privacy boundaries allow. Keep `list_spendable_assets` and cash balance
projection asset-only. Add conditional voucher claim listing and authority right
inventory listing. Add durable, family-aware, policy-aware quarantine that
survives restart and restore.
</action>
<acceptance_criteria>
- Wallet DB has a typed inventory facade and does not store vouchers or rights
  in `OwnedAssetPayload`.
- Existing `OwnedAssetPayload` rows with version 1 decode unchanged.
- Spendable cash listing includes only assets in spendable asset states.
- Voucher listing exposes offered, pending_accept, accepted, rejected,
  redeemable, partially_redeemed, redeemed, refunded, expired, quarantined
  states without adding to cash balance.
- Right listing exposes granted, held, delegated, consumed, revoked, expired,
  challenged, quarantined states without adding to value.
- Unknown policy quarantines are durable and family-aware.
- Migration tests cover old asset rows, new object rows, checksum mismatch, and
  status transition rejection.
</acceptance_criteria>

<threat_model>
- Reusing `OwnedAssetPayload` for vouchers or rights can make conditional claims
  or authority look like spendable cash.
- Non-durable quarantine lets restart bypass policy uncertainty.
- Rewriting all old asset rows without reversible migration can cause wallet
  data loss.
</threat_model>

<verification>
This plan passes only when wallet persistence has typed object inventory,
asset-only cash projection, durable quarantine, and old asset row compatibility.
</verification>
