<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| payment-request RPC → wallet.tx recipient selection | A stale or cross-chain request must not become a wallet.tx recipient. |
| session guard → sensitive wallet actions | Expired or locked sessions must fail closed before secrets are touched. |
| rotate_master_key RPC → audit log | The rotation path must never omit the audit trail or imply a persisted seed rewrite. |

| Threat ID | Category | Component | Disposition | Mitigation Plan |

| T-046-11 | D | `wallet_service_session_runtime_limits.rs` | mitigate | Reject over-limit or stale sessions early so the wallet does not spend work on requests that must fail. |
| T-046-12 | R | `server_admin.rs` | mitigate | Audit every master-key rotation attempt and keep the result limited to the in-memory rederive boundary. |
