<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| export pack -> restore validation | Missing manifest or duplicate validation could import inconsistent wallet state. |
| staged files -> live promotion | Partial promotion would corrupt wallet state across `.wlt` and JSONL planes. |
| legacy Snapshot input -> new owned-asset store | A compatibility bridge could accidentally keep Snapshot as live authority. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-047-19 | T | export/restore contract | mitigate | Add manifest, duplicate validation, and explicit JSONL-plane markers. |
| T-047-20 | D | staged promotion | mitigate | Keep all-or-nothing staging and wrong-password/corrupt-history tests. |
| T-047-21 | R | Snapshot compatibility | mitigate | Limit it to one-shot legacy input or remove it entirely from normal live paths. |
</threat_model>
