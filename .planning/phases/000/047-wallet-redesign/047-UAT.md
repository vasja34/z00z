## Tests

### 1. Cold Start Smoke Test

expected: From a cold start with no warm in-memory wallet state, creating or opening the wallet should succeed without startup errors, the encrypted `.wlt` storage should become the live authority immediately, and a basic wallet query should return live data without depending on Snapshot-only state.
result: [pending]

### 2. Profile-Only Save And Reopen

expected: Creating a wallet, saving it, closing it, and reopening it should restore profile data from `WalletProfilePayload`; a normal non-asset save should not require or rewrite a Snapshot claimed-assets sidecar.
result: [pending]

### 3. YAML-Backed Wallet Defaults

expected: Wallet create, open, recover, and backup flows should honor YAML-backed defaults or explicit overrides for auto-lock, backup location, and recovery gap limit instead of silently falling back to hardcoded literals.
result: [pending]

### 4. Receive Scan Persistence And Replay Idempotency

expected: Receiving or scanning a range should persist discovered owned assets and `ScanStatePayload` in one `.wlt` transaction, and replaying the same chunk after restart or cursor reset should not duplicate the owned assets.
result: [pending]

### 5. Owned-Asset Transaction Lifecycle

expected: `wallet.tx.*` build, cancel, import, and reconcile flows should use owned-asset authority for balances and pending state, prevent double-selection of pending inputs, and leave the wallet coherent after cancel or reconcile.
result: [pending]

### 6. Explicit Backup Export Boundary

expected: Backup or export should produce a manifest-backed wallet pack with profile, owned assets, scan state, and an explicit `wallet_<stem>_tx_history.jsonl` history plane marker; live tx history should remain a JSONL sidecar rather than moving into `.wlt`.
result: [pending]

### 7. Restore And Tamper Protection

expected: `WalletPlusHistory` restore should stage a fresh `.wlt` plus JSONL sidecar, restore identical owned assets and tx history on success, and reject tampered or mixed-authority imports without mutating live wallet state.
result: [pending]

### 8. Stage 13 Executed Lifecycle Proof

expected: Scenario 1 Stage 13 should execute the live wallet tx lifecycle over persisted owned assets, prove backup-restore, reopen, and tamper behavior for real, and keep simulator wording honest about `.wlt` owned assets plus canonical JSONL history instead of Snapshot authority.
result: [pending]
