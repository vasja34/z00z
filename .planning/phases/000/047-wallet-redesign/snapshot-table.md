## Review Rules

1. Every row must classify whether `snapshot` is live domain vocabulary, neutral technical vocabulary, or legacy wallet-snapshot residue.
2. `Recommendation` must use one of: `Keep`, `Remove`, `Rename`, `Migrate`, `Defer`.
3. `Comments` must explicitly state whether the recommendation was doublechecked.
4. Recommendations are Phase 047 specific: the question is whether the usage is invalid after the wallet redesign, not whether the word `snapshot` is globally forbidden.
5. If `Recommendation` is `Rename`, `Comments` must include a concrete replacement target.
6. If stale snapshot naming belongs to a dead or test-only API surface with no live consumer, prefer `Remove` over `Rename`.
