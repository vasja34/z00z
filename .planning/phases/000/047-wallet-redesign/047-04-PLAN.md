<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| owned-asset object -> duplicate handling | Bad duplicate handling would create multiple truths for one `asset_id`. |
| service wrappers -> store authority | Compatibility methods could silently keep mutating the old cache or Snapshot path. |
| asset catalog -> balance UX | User-facing asset lists must not read a stale mirror after cutover. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-047-10 | T | duplicate asset inserts | mitigate | Enforce idempotent-only duplicates and same-`object_id` updates. |
| T-047-11 | I | service wrappers | mitigate | Route public compatibility methods through the new store only. |
| T-047-12 | R | asset catalog | mitigate | Cut catalog reads to `OwnedAssetPayload` and migrate tests in the same wave. |
</threat_model>
