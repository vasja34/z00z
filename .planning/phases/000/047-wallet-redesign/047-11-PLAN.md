<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| remote worker -> wallet receive lane | Remote infrastructure can supply data, but it must not become the ownership authority. |
| worker hints -> cursor and asset writes | Resume or proof hints can corrupt local state if accepted without local scanner validation. |
| degraded network -> fallback behavior | Partial worker output must not leave cursor and owned-asset state half-advanced. |

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-047-31 | S | worker contract | mitigate | Keep the worker API evidence-only and forbid any direct ownership or mutation claim. |
| T-047-32 | T | cursor and asset persistence | mitigate | Preserve `recv_range(...)` as the only state-transition authority and reject out-of-band worker mutations. |
| T-047-33 | D | fallback and replay handling | mitigate | Add deterministic fallback plus replay/desync tests so partial worker output never advances state silently. |
</threat_model>
