# 058-01 Plan: Evidence Ledger And Source-Honest Acceptance Map

<objective>
Build the cross-phase evidence ledger required for Phase 058 and freeze the
live-versus-successor-versus-proposed acceptance map before later slices claim
readiness.

Purpose: make every later closeout row auditable instead of descriptive.
Output: one machine-readable and human-readable evidence ledger plus one honest
path-classification layer for tests, benches, configs, manifests, and reports.
</objective>

<action>
Create the one evidence ledger that later slices must fill instead of
rephrasing. For every TODO claim, fixture family, evidence-gap class,
design-artifact row, and final verdict category, record the exact owner home,
the exact command or regeneration entrypoint, the exact artifact path, the
digest or result expectation, and the current verdict state. Where the TODO
names an artifact or test that is not yet an exact live repo path, classify it
as `successor live` or `proposed` instead of silently treating it as landed.
Do not create a second evidence, benchmark, fixture, or acceptance owner when
an existing live owner home already exists in the repository.
Keep the Appendix C rows literal by mapping the one-machine multi-aggregator
simulation plan, deployment-shape notes, local-versus-replicated journal
notes, RAID-like topology notes, implementation-options notes, and benchmark
templates to checked artifacts or explicit gaps instead of collapsing them into
one generic documentation row.
</action>
