# Phase 058 Evidence Ledger

## 🎯 Purpose

This is the canonical Phase 058 claim ledger. It freezes one auditable row
shape for gates, fixture families, design artifacts, evidence-gap classes, and
final-verdict vocabulary.

Every accepted row in this file resolves to:

- one exact repository command;
- one exact artifact home;
- one digest or result set;
- one explicit verdict.

Rows that do not yet satisfy that bar stay `partial`, `open`, or
`unsupported`; design wording does not auto-promote them into live repository
facts.

## 🧭 Vocabulary

| Field | Meaning |
| --- | --- |
| `home_status` | `verified live` = exact file already exists; `successor live` = the exact TODO filename is absent but a checked live owner home already covers the seam; `proposed` = no exact live owner home is frozen yet. |
| `evidence_status` | `closed` = exact command, artifact, and verdict are already frozen; `partial` = live anchors exist but the full Phase 058 closeout packet is not complete yet; `open` = the row is still owned by a later slice; `unsupported` = the phase must not imply this claim today. |
| `digest_or_result_set` | The exact digest family, fixture ids, report rows, or bounded success conditions that the command must prove. |

## 🔒 Canonical Rules

- `058-TODO.md` plus the referenced HJMT docs are live Phase 058 scope
  authority.
- They are not permission to claim an exact repository path exists when the
  path is still TODO-only.
- `crates/z00z_storage/outputs/settlement/` is the current live measured bench
  home. `crates/z00z_storage/outputs/assets/` is retired planning wording, not
  a second archive path and not a remaining requirement.
- `SIM-5A7S` and `SIM-5A7S-PUB` remain the canonical acceptance fixtures.
  Phase 058 must still prove at least one additional positive non-`5x7`
  topology before any integrated or release-ready claim can stand.
- The inherited trace lineage `cfg_flow.json` through `recovery_flow.json`
  remains authoritative. Additional trace files stay explicit until their exact
  homes land.
